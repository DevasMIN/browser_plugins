# YouTube Deep Feed

Собственная бесконечная хронологическая лента по всем подпискам YouTube — вместо
штатной ленты `/feed/subscriptions`, которую YouTube обрезает (ограниченное окно,
~100 видео на канал, капризы бэкенда).

## Что умеет

- **Запас вглубь.** У каждого канала поддерживается буфер из ~20 непросмотренных
  видео: посмотрели — синхронизация докачает следующие, более старые. Всё хранится
  локально (IndexedDB), уже скачанное не перекачивается.
- **Просмотренное не показывается.** Отметки берутся из самого YouTube:
  из истории просмотра и из полосок прогресса на превью (порог — 90%).
  Клик по видео в ленте тоже помечает его просмотренным (возвращается кнопкой ↩).
- **«Только новые» для канала.** В панели «Каналы» кнопка «✓ только новые»
  ставит каналу водяной знак: ВСЕ его видео старше момента нажатия считаются
  просмотренными — включая те, что ещё не скачаны в базу. Докачка старых
  выключается, в ленте от канала остаются только свежие видео. Явная отметка
  «↩ не смотрел» на конкретном видео сильнее водяного знака. Кнопка
  «⟲ и старые» снимает знак и возвращает докачку.
- **Скрытие.** Отдельное видео («✕ скрыть») или канал целиком (панель «Каналы»).
  Если видео есть в нативной ленте YouTube (у него сохранён feedback-токен из
  синхронизации), «✕ скрыть» скрывает его и там.
  При скрытии внизу появляется тост «Отменить» (8 сек) — на случай случайного
  клика; отмена возвращает карточку и, если скрывали на YouTube, снимает скрытие
  и там.
- **Смотреть позже.** Кнопка «⏳ позже» добавляет видео в ваш плейлист
  «Смотреть позже» на YouTube и убирает его из этой ленты (тоже с отменой).
  Видео, уже лежащие в «Смотреть позже», в ленте не показываются — список WL
  подтягивается с YouTube и потому общий для всех устройств.
- **Автообновление.** Пока вкладка открыта, лента сама подтягивает новое:
  лёгкая проверка нативной ленты каждые 15 минут, полный обход каналов раз в
  6 часов. Если вы вверху — новые карточки появляются сразу; если проматали
  вниз — сверху всплывает кнопка «↑ N новых». Нативная лента (`FEsubscriptions`)
  заодно даёт свежие отметки прогресса, а видео, которое по дате попадает в окно
  ленты, но в ней отсутствует, помечается скрытым — так подхватываются ваши
  «Скрыть» с YouTube (дифф включается только на здоровой ленте: ≥100 видео и
  окно ≥7 дней).
- **Экспорт / импорт.** Кнопки «Экспорт» и «Импорт» сохраняют и загружают всю
  базу и настройки (каналы, видео, отметки просмотра, скрытое, «только новые»)
  одним JSON-файлом. Это и резервная копия, и надёжный перенос между
  устройствами: экспортировал на одном, импортировал на другом. Работает в
  любом браузере, ничего не может «слететь».
- **Облачная синхронизация скрытого между устройствами.** Скрытое (ручное,
  «Смотреть позже») зеркалится через `chrome.storage.sync` — облачное хранилище
  браузера. На всех устройствах, где стоит это расширение и включена
  синхронизация браузера под одним аккаунтом (Opera Sync), список скрытого
  общий. (Автоматически распознанные из ленты «feed-diff» не синхронизируются —
  они пересчитываются на каждом устройстве и экономят квоту sync ~100 КБ.)

  Для этого у расширения должен быть **один и тот же ID** на всех устройствах,
  иначе `chrome.storage.sync` разложит данные по разным «ящикам». ID зафиксирован
  через манифест `key` (публичный ключ), поэтому при загрузке этой же папки на
  любой машине ID одинаковый. **Внимание:** `key` фиксирует ID, но менять его
  задним числом нельзя — смена ID отвязывает локальную базу (IndexedDB) и
  настройки «слетают». Перед любым изменением ID делайте Экспорт.

  **Про `key` для тех, кто ставит из этого репозитория:** генерировать ничего
  не нужно — оставляйте как есть (это публичный ключ, общий ID ничем не грозит:
  данные каждого пользователя живут в его браузере). Не нужен фиксированный
  ID — просто удалите строку `"key"` из manifest.json. Нужен свой — сгенерируйте:
  `openssl genrsa 2048 | openssl rsa -pubout -outform DER | base64 -w0`
  и вставьте результат в `"key"` (менять его после начала работы нельзя,
  см. выше).
- Поиск по названию, фильтр по каналу, бесконечная прокрутка.

## Установка

1. `chrome://extensions` → включить «Режим разработчика».
2. «Загрузить распакованное расширение» → выбрать папку `youtube_deep_feed`.
3. Быть залогиненным на youtube.com в этом же профиле Chrome.
4. Клик по иконке расширения → страница ленты → «Начать первичную синхронизацию».

## Порядок первого запуска

1. **Синхронизация** — список подписок + свежие видео каждого канала
   (~2 минуты на ~270 каналов). Дальше запускается кнопкой или автоматически
   при открытии ленты (раз в 6 часов) и заодно подкачивает старые видео до
   запаса в ~20 непросмотренных на канал (прогресс вглубь хранится per-канал
   и переживает прерывания).
2. **История** — импорт отметок «просмотрено» из истории YouTube (возобновляемый:
   можно запускать повторно, продолжит с места остановки).

## Как это работает

Страница расширения ходит в внутренний API YouTube (`/youtubei/v1/browse`)
с куками youtube.com и подписью `SAPISIDHASH` — ровно так же, как сам сайт.
Если Chrome не отдаёт куки со страницы расширения (HTTP 403), запросы
автоматически идут через реле — контент-скрипт во вкладке youtube.com
(при необходимости открывается закреплённая вкладка).
Каналы читаются из `FEchannels`, архив канала — из плейлиста загрузок
`UULF…` (без Shorts; если пуст — `UU…`), история — из `FEhistory`.

## Ограничения

- Даты публикации у YouTube отдаются относительными («2 года назад»), поэтому
  сортировка старых видео приблизительная: с точностью до дня для свежих,
  до месяцев для многолетней давности. Порядок видео внутри канала точный.
- «Скрыть» в этой ленте передаётся на YouTube только для видео, которые побывали
  в нативной ленте (у них есть feedback-токен). В обратную сторону «Скрыть»,
  сделанные на самом YouTube, подхватываются автоматически через дифф ленты
  (см. «Автообновление»).
- Chrome/Opera (Manifest V3). Версии для Firefox нет.
